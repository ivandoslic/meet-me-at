# Architecture Decision Records

ADRs capture decisions that materially affect system structure or engineering
practice.

Status values:

- Proposed
- Accepted
- Superseded
- Rejected

When a decision changes, add a new ADR and mark the earlier record as
superseded. Do not rewrite history to make the original decision look different.

## Template

```text
# ADR-NNN: Title

Status: Proposed
Date: YYYY-MM-DD

## Context
## Decision
## Alternatives considered
## Consequences
```
