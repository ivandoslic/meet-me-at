# Documentation

- `architecture/` explains system boundaries and data flow.
- `decisions/` records decisions and their consequences.
- `conventions/` defines cross-repository engineering rules.
- `guides/` provides milestone implementation guidance.
- `domain-glossary.md` defines shared product language.
- `roadmap.md` defines the intended sequence of delivery.

Documentation describes the current intended architecture. If implementation
proves a decision wrong, update both the implementation and the corresponding
documentation.
